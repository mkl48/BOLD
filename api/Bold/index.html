Bold

The chat library's entry point. Require it on both realms and call Bold.Start. Everything tunable flows through Bold.Configure; messages go out via Bold.Message or the fluent Builder from Bold.Compose.

Functions​

Configure​

</>
Bold.Configure(config: Types.BoldConfig?) → ()

Deep-merges a [BoldConfig] into the live config. Safe before or after Bold.Start. Sections: interface, parsing, filter, keys, animation, sounds.

Theme​

</>
Bold.Theme(preset: string) → boolean

Applies a built-in theme preset (dark, light, midnight, neon). Returns false if the name is unknown.

Layout​

</>
Bold.Layout(preset: string) → boolean

Applies a window layout preset (top_left, top_right, bottom_left, bottom_right, center). Returns false if the name is unknown.

Start​

</>
Bold.Start(config: Types.BoldConfig?) → ()

Boots Bold on the current realm. On the server it wires the receive → filter → broadcast pipeline; on the client it builds the UI and input. Accepts an optional [BoldConfig] (the same shape Bold.Configure takes). Call once per realm.

Message​

</>
Bold.Message(config: Types.MessageClass?) → Types.MessageClass

Constructs and sends a message. On the server it broadcasts, or whispers when metadata.recipient_id is set; on the client it goes through the send pipeline.

Compose​

</>
Bold.Compose() → Builder

Returns a fluent Builder for assembling a message from markdown + embeds, e.g. Bold.Compose():Bold("gg"):Mention(plr):Send().

Dispatch​

This item only works when running on the server. Server
</>
Bold.Dispatch(
config: Types.MessageClass?,
target: Player?
) → Types.MessageClass?

Sends a server-originated message without routing through the client SEND channel. Pass target to whisper it to one player.

Image​

</>
Bold.Image(
image_id: string,
config: Types.ImageEmbed?
) → Types.ImageEmbed

Builds an Image embed for a message's content.embeds.

Gif​

</>
Bold.Gif(
video_id: string,
config: {
loop: boolean?,
autoplay: boolean?,
color: Color3?
}?
) → Types.GifEmbed

Builds a Gif (VideoFrame) embed for a message's content.embeds.

Sticker​

</>
Bold.Sticker(
sticker_id: string,
config: {
size: number?,
color: Color3?
}?
) → Types.StickerEmbed

Builds a Sticker embed for a message's content.embeds.

RichText​

</>
Bold.RichText(config: Types.RichTextEmbed) → Types.RichTextEmbed

Builds a RichText embed (title, description, fields) for a message's content.embeds.

Action​

</>
Bold.Action(
action_id: string,
config: {
title: string?,
button_text: string?,
style: ("primary" | "danger" | "ghost")?,
color: Color3?,
}?
) → Types.ActionEmbed

Builds an Action (button) embed. Handle clicks via the EmbedActionFired event.

Buttons​

</>
Bold.Buttons(
buttons: {Types.ButtonSpec},
config: {
title: string?,
color: Color3?
}?
) → Types.ButtonsEmbed

Builds a row-of-buttons embed. Handle clicks with Bold.OnAction.

Progress​

</>
Bold.Progress(
value: number,
config: {
label: string?,
color: Color3?
}?
) → Types.ProgressEmbed

Builds a progress-bar embed. value is 0..1.

On​

</>
Bold.On(
eventName: string,
callback: (...any) → ()
) → RBXScriptConnection

Connects to a Bold event by name (MessageReceived, Mentioned, ReactionAdded, LinkClicked, ThemeChanged, …). Returns the connection.

Once​

</>
Bold.Once(
eventName: string,
callback: (...any) → ()
) → RBXScriptConnection

Like Bold.On, but auto-disconnects after the first fire.

Hook​

</>
Bold.Hook(
hookName: string,
fn: (...any) → ()
) → ()

Registers an observational lifecycle hook (BeforeSend, AfterDeliver, BeforeRender, …). Fire-and-forget: it can observe but not alter or drop a message.

OnMentioned​

This item only works when running on the client. Client
</>
Bold.OnMentioned(fn: (message: any) → ()) → RBXScriptConnection

Fires when the local player is @mentioned in a delivered message.

OnAction​

</>
Bold.OnAction(fn: (
action_id: string,
message: any?
) → ()) → RBXScriptConnection

Fires when an Action/Buttons embed button (or a poll option) is clicked. action_id is the id you gave the button; poll options fire "poll:<option>".

Guard​

</>
Bold.Guard(fn: (
message: any,
player: Player?
) → (boolean?)) → ()

Installs a message guard for the current realm. Return false to drop a message before it is delivered (server) or rendered (client).

Classify​

</>
Bold.Classify(
message: Types.MessageClass,
kind: string,
tags: {string}?
) → Types.MessageClass

Sets a message's kind (and optional tags), firing the relevant events.

Tag​

</>
Bold.Tag(
message: Types.MessageClass,
...: string
) → Types.MessageClass

Adds one or more tags to a message, de-duplicated.

Pin​

</>
Bold.Pin(message: Types.MessageClass) → Types.MessageClass

Pins a message card (adds the pinned indicator).

Unpin​

</>
Bold.Unpin(message: Types.MessageClass) → Types.MessageClass

Unpins a message card.

React​

This item only works when running on the client. Client
</>
Bold.React(
message: any,
emoji: string
) → ()

Toggles the local player's reaction (an emoji) on a message. Accepts a message object or a message id.

RegisterEmoji​

</>
Bold.RegisterEmoji(
name: string,
assetId: string
) → ()

Registers a custom :name: emoji backed by an image asset id.

RegisterSticker​

</>
Bold.RegisterSticker(
id: string,
assetId: string
) → ()

Registers a sticker id → asset mapping used by sticker embeds and autocomplete.

SearchEmoji​

</>
Bold.SearchEmoji(query: string) → {string}

Searches built-in + custom emoji by partial name.

RegisterCommand​

</>
Bold.RegisterCommand(
name: string,
spec: Types.CommandSpec
) → ()

Registers a slash /command. The handler returns a compose payload ({ text, kind?, recipient?, embeds? }) to send, or nothing for a local-only command. Built-ins: /me, /shrug, /clear, /help, /incog.

RegisterPanel​

This item only works when running on the client. Client
</>
Bold.RegisterPanel(
name: string,
fn: (
args: any,
message: any,
theme: any
) → Frame
) → ()

Registers a renderer for a :::name … ::: panel fence. fn(args, message, theme) returns a Frame. Unregistered names render a graceful placeholder.

AddBlockedWords​

</>
Bold.AddBlockedWords(words: {string} | string) → ()

Adds words to the local [Advise] blocklist (layered under Roblox's TextService filter). The bulk list is loaded from Proflis; this is for one-off additions.

AddAllowedWords​

</>
Bold.AddAllowedWords(words: {string} | string) → ()

Allow-lists words so a blocked substring inside them is never masked (Scunthorpe-style exceptions), e.g. Bold.AddAllowedWords("assassin").

SetTheme​

This item only works when running on the client. Client
</>
Bold.SetTheme(theme: Types.Theme) → ()

Merges theme token overrides into the live theme (see also Bold.Theme presets).

SetDensity​

This item only works when running on the client. Client
</>
Bold.SetDensity(d: "compact" | "cozy" | "comfortable") → ()

Sets the message row density.

FocusInput​

This item only works when running on the client. Client
</>
Bold.FocusInput() → ()

Focuses the chat input box.

SetKindStyle​

This item only works when running on the client. Client
</>
Bold.SetKindStyle(
kind: string,
textKey: string,
prefix: string?,
bold: boolean?
) → ()

Restyles a message kind's prefix, text-colour token and bold flag.

SetKindOverride​

This item only works when running on the client. Client
</>
Bold.SetKindOverride(
kind: string,
bgKey: string,
textKey: string
) → ()

Overrides the background/text theme keys used for a message kind's card.

Decorate​

This item only works when running on the client. Client
</>
Bold.Decorate(fn: (any) → string?) → ()

Registers a per-row decorator that appends inline rich text; fn(message) returns a string (or nil to add nothing).

SetChannelName​

This item only works when running on the client. Client
</>
Bold.SetChannelName(name: string) → ()

Sets the header/channel label.

Toggle​

This item only works when running on the client. Client
</>
Bold.Toggle(visible: boolean?) → ()

Shows or hides the chat window (or forces a state).

Clear​

This item only works when running on the client. Client
</>
Bold.Clear() → ()

Clears the local message list.

RegisterRenderer​

This item only works when running on the client. Client
</>
Bold.RegisterRenderer(
kind: string,
fn: (
any,
any
) → Frame
) → ()

Registers a fully custom card renderer for a message kind; fn(message, theme) returns a Frame used instead of the default row.
